## 목적

사용자가 영상 아이디어를 가져오면, 처음부터 끝까지 완성된 기획서를 만들 수 있도록 단계별로 안내합니다.
장르, 타겟, 씬 구성, 기법 추천, 최종 기획서 저장까지 전 과정을 커버합니다.

## 사용 시점

- 사용자가 `/first-cut`을 호출했을 때
- "영상 기획해줘", "유튜브 영상 만들고 싶어", "영상 콘텐츠 아이디어가 있는데" 등의 요청이 올 때
- 새 영상 프로젝트를 처음 시작할 때

---

## 워크플로우 (8단계)

### 0단계: 세션 재개 확인

`plans/drafts/` 디렉토리를 확인하세요.
이전에 진행 중이던 기획 파일(예: `{프로젝트명}-step*.md`, `{프로젝트명}-scene*.md`)이 있으면 사용자에게 물어보세요:

> "이전에 진행 중이던 기획이 있습니다: [파일명]. 이어서 진행할까요, 아니면 새로 시작할까요?"

이어서 진행하는 경우, 해당 파일들을 읽어 현재까지의 진행 상태를 파악한 후 이어서 시작하세요.

---

### 1단계: 아이디어 수집

사용자의 영상 제작 아이디어를 자유롭게 받으세요.
짧은 한 마디도 괜찮습니다. 처음부터 완벽하지 않아도 됩니다.

예시 질문:
> "어떤 영상을 만들고 싶으신가요? 주제나 아이디어를 자유롭게 말씀해 주세요."

---

### 2단계: 사전 정보 수집 (순차 질문)

아래 항목을 **하나씩** 순서대로 질문하세요. 한 번에 몰아서 묻지 마세요.

1. **장르**: "어떤 종류의 영상인가요? (유튜브 브이로그, 교육/튜토리얼, 광고, 숏폼, 다큐멘터리, 뮤직비디오 등)"
2. **타겟 오디언스**: "주로 누가 볼 영상인가요? (연령대, 관심사, 구독자 특성 등)"
3. **예상 길이**: "총 몇 분짜리 영상을 생각하시나요?"
4. **분위기**: "어떤 느낌의 영상을 원하시나요? (밝고 경쾌한, 감성적인, 전문적인, 유머러스한 등)"
5. **레퍼런스**: "참고하고 싶은 영상이나 채널이 있나요? (없어도 괜찮습니다)"
6. **촬영 장비 수준**: "어떤 장비로 촬영하실 예정인가요? (스마트폰, 미러리스 카메라, 전문 장비 등)"

각 답변을 받을 때마다 자연스럽게 이어지도록 반응하며 다음 질문으로 넘어가세요.
수집이 완료되면 `plans/drafts/{프로젝트명}-step1.md`에 수집된 정보를 저장하세요.

---

### 3단계: 벤치마크 및 트렌드 조사

수집된 정보를 바탕으로 `.claude/skills/benchmark/SKILL.md`의 지침을 따라 리서치를 수행하세요.

- 웹 검색 도구(WebSearch)를 사용하여 유사 콘텐츠와 현재 트렌드를 조사하세요
- 조사 결과를 사용자에게 요약하여 공유하세요
- 결과를 `plans/drafts/{프로젝트명}-benchmark.md`에 저장하세요

---

### 4단계: 전체 구조 제안

조사 결과와 수집된 정보를 바탕으로 영상의 전체 구조를 제안하세요.

제안 내용:
- 총 씬(장면) 개수 및 각 씬의 역할
- 전체 흐름과 감정 곡선
- 예상 씬별 길이 배분

사용자 피드백을 받아 구조를 확정하고 `plans/drafts/{프로젝트명}-step2.md`에 저장하세요.

---

### 5단계: 씬별 상세 구체화 (대화형)

각 씬을 사용자와 함께 하나씩 대화하며 구체화하세요.
`.claude/skills/scene-builder/SKILL.md`의 지침을 따르세요.

각 씬마다 아래 기법 스킬의 결정 매트릭스를 참고하여 추천하세요:

- **촬영 기법**: `.claude/skills/cinematography/SKILL.md`의 결정 매트릭스를 참고하세요
- **사운드**: `.claude/skills/sound-design/SKILL.md`의 결정 매트릭스를 참고하세요
- **편집 기법**: `.claude/skills/editing/SKILL.md`의 결정 매트릭스를 참고하세요
- **연출**: `.claude/skills/directing/SKILL.md`의 결정 매트릭스를 참고하세요
- **색보정**: `.claude/skills/color-grading/SKILL.md`의 결정 매트릭스를 참고하세요
- **장르 특성**: `.claude/skills/genre-guide/SKILL.md`의 매핑 테이블을 참고하세요

씬 완성 시마다 `plans/drafts/{프로젝트명}-scene{N}.md`에 저장하세요.

---

### 6단계: 초안 저장

모든 씬이 완성되면 `plans/drafts/{프로젝트명}-최종초안.md`에 전체 초안을 통합 저장하세요.

---

### 7단계: 최종 확인 및 기획서 생성

사용자에게 초안을 검토할 기회를 주세요:
> "초안을 완성했습니다. 검토 후 수정할 내용이 있으면 알려주세요. 확정이 되면 최종 기획서를 생성하겠습니다."

수정 사항을 반영한 후 `.claude/skills/export-plan/SKILL.md`의 지침을 따라 최종 기획서를 생성하세요.
파일명: `plans/YYYY-MM-DD-{제목}.md` (예: `plans/2026-03-30-요리영상기획.md`)

---

### 8단계: 노션 동기화 안내

기획서 완성 후 사용자에게 물어보세요:
> "노션에 기획서를 동기화하시겠습니까? `/notion-sync`를 호출하면 노션 페이지로 저장할 수 있습니다."

---

## 중간 저장 파일 명명 규칙

| 파일 | 저장 시점 |
|------|-----------|
| `plans/drafts/{프로젝트명}-step1.md` | 사전 정보 수집 완료 시 |
| `plans/drafts/{프로젝트명}-benchmark.md` | 벤치마크 조사 완료 시 |
| `plans/drafts/{프로젝트명}-step2.md` | 전체 구조 확정 시 |
| `plans/drafts/{프로젝트명}-scene{N}.md` | 각 씬 완성 시 |
| `plans/drafts/{프로젝트명}-최종초안.md` | 전체 씬 완성 시 |
| `plans/YYYY-MM-DD-{제목}.md` | 최종 확정 시 |

---

## 용어 설명

- **씬(Scene)**: 영상을 구성하는 하나의 장면 단위. 장소나 시간이 바뀌면 새로운 씬이 됩니다
- **감정 곡선**: 영상 전체에서 시청자의 감정이 어떻게 변해가는지 나타내는 흐름
- **벤치마크**: 비슷한 영상을 분석하여 참고 기준을 세우는 작업
